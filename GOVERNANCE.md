# Governance

PyMixEF is designed for public technical governance.

- A technical steering committee should include numerical statistics,
  pharmacometrics, clinical biostatistics, scientific software, validation, and
  user representation.
- Model semantics, IR schema, result contracts, and estimator changes use a
  public request-for-comments process.
- Numerical kernels require two reviewers, including one reviewer with the
  relevant statistical or pharmacometric expertise.
- Conflicts of interest and material funding are disclosed.
- The Python API follows semantic versioning; the model IR has its own schema
  version and migration policy.
- Stable and reference-validated labels are evidence gates, not schedule labels.
- The core remains usable offline and has no telemetry.

The initial repository is a technical implementation, not a claim that the
long-term steering committee or independent scientific review has already been
constituted.
